Creare report

Le relazioni sono documenti formattati secondo un determinato modello di informazioni contenute in un'analisi, uno scenario o una determinata scheda.

I rapporti di modifica dell'interfaccia sono accessibili su ogni riga della tabella Riepilogo delle valutazioni in una determinata analisi o scenario.

Interfaccia di segnalazione

L'interfaccia della relazione si compone delle tre componenti principali seguenti:

  • la barra degli strumenti Modifica/esportazione nella parte superiore dell'interfaccia consente di modificare il rapporto e di selezionare un formato per le sue esportazioni (json, XLSX, PDF o DOCX),
  • la barra degli strumenti di filtraggio delle informazioni consente di selezionare le informazioni da segnalare, in base alla loro identificazione o valutazione,
  • il riquadro Anteprima fornisce una panoramica delle informazioni contenute nelle relazioni finali. Fornisce anche un semplice editor per aggiungere o rimuovere informazioni nel rapporto. Si noti che il rendering finale può variare a seconda del formato di esportazione selezionato.

Modifica/esportazione barra degli strumenti

La barra degli strumenti Modifica/Esportazione per la modifica delle informazioni delle relazioni e per l'esportazione in un determinato formato:

Aprire un semplice editor nella cornice di anteprima per aggiungere o rimuovere informazioni nel report generato. Si noti che le informazioni modificate saranno visibili solo quando si esporta in pdf o docx. La selezione di nuovi filtri dalla barra degli strumenti di filtraggio delle informazioni cancella tutte le modifiche correnti.
Esporta le informazioni grezze di una determinata analisi, scenario o scheda utilizzando il formato JavaScript Object Notation (JSON).
Esporta le informazioni grezze di una determinata analisi, scenario o scheda in fogli di calcolo utilizzando il formato Office Open XML (XLXS).
Esporta le informazioni formattate di una determinata analisi, scenario o scheda in formato PDF (Portable Document Format).
Esporta le informazioni formattate di una determinata analisi, scenario o scheda in un Office Open XML (DOCX) modificabile.

Le informazioni formattate corrispondono alla generazione di una rappresentazione HTML di un report basata su un modello in formato PUG. I nuovi modelli possono essere importati nella sezione Modello della barra di navigazione laterale sinistra e selezionati nella barra degli strumenti di filtraggio delle informazioni.

Barra degli strumenti di filtraggio delle informazioni

La barra degli strumenti di filtraggio delle informazioni consente di selezionare analisi, scenari e schede in base alla loro identificazione o alla rispettiva valutazione in base alle opzioni o a un modello:

Seleziona un modello Usare un modello PUG è stato importato dalla sezione Template della barra di navigazione laterale sinistra.
Seleziona i tag da segnalare Quando si riporta un'analisi completa, consente di selezionare una serie di scenari o tutti gli scenari da includere nel report.
Seleziona le carte da segnalare Quando si segnala un'analisi o uno scenario, consente di selezionare un set di carte o tutte le carte da includere nel rapporto.
Selezionare le valutazioni da segnalare Quando si riporta un'analisi, uno scenario o una scheda, consente di selezionare valutazioni o commenti da includere nel rapporto. Si noti che questa opzione si applica anche alle linee in cookie, archiviazione locale, beacon e testsl.

I modelli e le opzioni di filtraggio sono reciprocamente inclusivi. In quanto tali, i modelli possono essere informazioni complete su analisi, scenari e schede che potrebbero essere successivamente selezionate attraverso le opzioni di filtraggio.

Modifica di nuovi modelli

Tuttavia, l'unica sintassi supportata per il modello è PUG. I report generati da un modello corrispondono all'oggetto dati contenuto nel json di un'analisi, uno scenario o una scheda ai locali corrispondenti nella sintassi del carlino.

Puoi fare riferimento alla sezione Language Reference del suo sito ufficiale per ottenere informazioni complete nella sua sintassi.

IldescrizioneIl modello fornisce una serie di azioni per importare, modificare o duplicare i modelli esistenti:

Importare un modello nella sintassi PUG con un nome e un autore per il suo uso successivo nei report.
Rispettivamente scaricare ed eliminare i modelli importati

Come prima introduzione per la generazione di nuovi modelli, si consiglia di scaricare un'analisi, uno scenario o un tag dalla barra degli strumenti Modifica / esportazione in formato raw JSON per trovare le voci corrispondenti.

Molti Online Pug Testing Tool consentono di copiare / incollare il modello PUG e le informazioni JSON per rendere l'HTML finale. Il modello predefinito può anche essere esportato come riferimento.
