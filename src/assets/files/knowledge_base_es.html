Base de conocimientos

La información almacenada en los navegadores de un sitio web puede tener múltiples propósitos, ya sea para el correcto funcionamiento del sitio web o para fines subsidiarios, como el seguimiento de la navegación del usuario para entregar anuncios específicos. Como tal, algunos pueden requerir el consentimiento antes de su uso por parte del sitio web, o de terceros, para cumplir con los requisitos legales, mientras que otros podrían estar exentos.

Muchos de estos rastreadores se almacenan a través de SDK externos desarrollados por empresas de terceros, ya que comparten propiedades comunes entre sitios web, como el dominio, el nombre de la clave o los scripts.

Las bases de conocimiento pueden ayudar a identificar estos rastreadores, así como el caso consolidando el conocimiento a lo largo del tiempo durante el análisis caso por caso.

La identificación de rastreadores conocidos se produce durante la navegación seleccionando una cookie o una línea clave de almacenamiento local de las tarjetas correspondientes a través del panel del registrador del navegador interno.

Esta misma información también está disponible en Más detalles al evaluar el cumplimiento legal de un análisis. El panel de expansión Agregar este elemento a una base de conocimientos también permite agregar rastreadores aún desconocidos para su evaluación posterior.

Finalmente, el contenido completo del conocimiento es accesible a través de en la categoría Editor en la barra de navegación del lado izquierdo.

Edición de una base de conocimientos

Cada base de conocimiento se identifica con un nombre y un autor. Se compone de un conjunto de entradas que pueden dirigirse a cookies o claves en el almacenamiento local.

De forma predeterminada, la herramienta de auditoría del sitio web no contiene ninguna base de conocimientos. Los usuarios pueden crear uno nuevo, seleccionar un nombre y la categoría apropiada (Cookie o Almacenamiento local).
También pueden importar bases de datos existentes en formato json que ya han sido creadas por otros a través de la herramienta de auditoría del sitio web.
Finalmente, pueden exportar todas las bases de datos contenidas en esta herramienta a través de un archivo de formato json, para compartir o archivar.

Las entradas de la base de datos también pueden gestionarse individualmente mediante el conjunto de botones , que permiten, respectivamente, visualizar, descargar, duplicar o eliminar la entrada seleccionada.

Uso de la base de conocimientos

Cada cookie o entrada en el almacenamiento local que se almacena durante una sesión de navegación se escanea automáticamente para encontrar entradas coincidentes en la base de conocimientos.

Los propósitos identificados de las entradas coincidentes están disponibles en la fila Categoría en las tablas de Cookie y Tarjetas de almacenamiento local, así como en la siguiente pestaña del panel de detalles al seleccionar una línea en las tablas Conocimiento.

Si la entrada actual se ha encontrado en al menos una base de conocimientos, esta pestaña enumera todas las entradas coincidentes en la base de conocimientos de acuerdo con su categoría.

Si se trata de una cookie, estas entradas se clasifican de la siguiente manera:

  • Coincidencia exacta: al menos una entrada corresponde al nombre y al dominio de la cookie,
  • Nombre del partido: al menos una entrada corresponde al nombre de la cookie con un dominio comodín (*),
  • Dominio de coincidencia: al menos una entrada corresponde al dominio de esta cookie.

Si se trata de una clave en el almacenamiento local, al menos una entrada coincide con la clave o el script responsable de esta entrada.
