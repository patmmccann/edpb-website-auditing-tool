Base di conoscenza

Le informazioni memorizzate nei browser di un sito Web possono avere molteplici scopi, sia per il corretto funzionamento del sito Web sia per scopi sussidiari come il monitoraggio della navigazione dell'utente per fornire annunci pubblicitari mirati. In quanto tali, alcuni potrebbero richiedere il consenso prima del loro utilizzo da parte del sito Web, o di terzi, per essere conformi ai requisiti legali, mentre altri potrebbero essere esentati.

Molti di questi tracker sono archiviati tramite SDK esterni sviluppati da società di terze parti, in quanto condividono proprietà comuni tra siti Web, come il dominio, il nome della chiave o gli script.

Le basi di conoscenza possono aiutare a identificare questi tracker, nonché a consolidare le conoscenze nel tempo durante l'analisi caso per caso.

L'identificazione dei tracker noti avviene durante la navigazione selezionando un cookie o una linea chiave di archiviazione locale dalle schede corrispondenti attraverso il pannello del logger del browser interno.

Queste stesse informazioni sono accessibili anche in Maggiori dettagli durante la valutazione della conformità giuridica di un'analisi. Il pannello di espansione Aggiungere questo elemento a una base di conoscenze consente anche di aggiungere tracker ancora sconosciuti per la loro valutazione successiva.

Infine, il contenuto completo della conoscenza è accessibile attraverso nella categoria Editor sulla barra di navigazione laterale sinistra.

Utilizzo della Knowledge Base

Ogni cookie o voce nella memoria locale che viene memorizzata durante una sessione di navigazione viene automaticamente scansionata per trovare voci corrispondenti nella knowledge base.

Gli scopi identificati delle voci corrispondenti sono disponibili nella riga Categoria nelle tabelle Cookie e Schede di archiviazione locali, nonché nella scheda seguente dal pannello dei dettagli quando si seleziona una riga nelle tabelle Conoscenza.

Se la voce corrente è stata trovata in almeno una base di conoscenza, questa scheda elenca tutte le voci corrispondenti nella base di conoscenza in base alla sua categoria.

Se si tratta di un cookie, queste voci sono classificate come segue:

  • Corrispondenza esatta: almeno una voce corrisponde al nome e al dominio del cookie,
  • Nome della corrispondenza: almeno una voce corrisponde al nome del cookie con un dominio jolly (*),
  • Corrispondenza dominio: almeno una voce corrisponde al dominio di questo cookie.

Se si tratta di una chiave nella memoria locale, almeno una voce corrisponde alla chiave o allo script responsabile di questa voce.

È possibile fare riferimento a Fornire nuove analisi per l'utilizzo di tali informazioni al fine di valutare la conformità / non conformità dei siti Web in base ai requisiti legali.
